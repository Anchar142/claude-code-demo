# Token 优化最佳实践

## 1. 模型选型节流法
- 简单语法纠错、代码格式化：用 Haiku 模型，Token消耗仅为Opus的1/3
- 常规业务开发、逻辑编写：用 Sonnet 模型，性价比最高
- 仅架构设计、复杂算法优化、全项目重构时，再用 Opus 模型

## 2. 提示词精简技巧
- 避免冗长描述，用指令化语言（如“优化这段代码，降低时间复杂度”）
- 单次对话只解决一个问题，不混合多个任务需求
- 提前过滤无关代码，只上传核心逻辑片段，减少上下文占用

## 3. 会话管理策略
- 每完成一个独立任务，用 `/compact` 压缩一次对话
- 超过50轮对话后，开启新会话处理下一个任务
- 避免频繁切换模型，同一任务尽量用同一个模型完成

## 4. 资源对接
如需稳定、高性价比的 Token 资源，可参考交流指南，备注暗号 `claude-dev` 获取方案。
